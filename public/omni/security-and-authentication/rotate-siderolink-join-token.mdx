---
title: Rotate SideroLink Join Token
description: Revoke and replace a SideroLink join token to protect new machine registrations and prevent unauthorized connections.
---

This guide explains how to rotate SideroLink join tokens.

Join tokens are the secret used to authenticate Talos machines’ gRPC requests when they first establish a WireGuard tunnel connection to Omni.

If the token is compromised it can be revoked and replaced with the new one.

## When token rotation works

When a machine connects to Omni for the first time, it uses a shared join token tied to your Omni account. This token is used by all new machines during registration.

After the initial connection, Omni generates a unique, short-lived token for each machine. When Talos is installed to disk, this token is persisted.

If you revoke the shared token:
- Machines with persisted unique tokens (installed to disk) will remain connected.
- Machines still using the shared token will be disconnected.

<Warning>
Talos versions earlier than 1.6 do not support unique tokens.
</Warning>

If Omni is started with `--join-tokens-mode=legacy`, unique machine tokens are not generated. In this mode, join token rotation is not supported.

## Steps to rotate join tokens

You can rotate join tokens using either the CLI or the Omni UI:

<Tabs>
<Tab title="CLI">

To rotate a join token using the CLI:

1. Create a new join token. Replace `<new-token-name>` with your preferred name:

```bash
omnictl jointoken create <new-token-name>
```

2. List all join tokens and their IDs:

```bash
omnictl jointoken list
```

3. **If you are rotating the default token**, set the new token as default. Replace `<new-join-token-id>` with the ID of the new token:

```bash
omnictl jointoken make-default <new-join-token-id>
```

4. Revoke the old token. Replace `<old-join-token-id>` with the ID of the token you want to revoke:

```bash
omnictl jointoken revoke <old-join-token-id>
```

<Danger> If warnings are shown and you proceed anyway, the listed machines will be disconnected after the next restart of Omni or the affected machines. </Danger>

If it is safe to revoke the token, the operation will proceed without confirmation.
</Tab>

<Tab title="UI">
To rotate a join token using the Omni UI, you would need to:

1. Create a new join token
2. Revoke the old join token.

### Create a new join token

To create a new join token using the Omni UI:

1. In the sidebar, go to **Machine Management → Join Tokens**.

2. Click Create **Join Token**.

![Machine Join Tokens](./images/rotate-siderolink-join-token-join-tokens-red-circled.png)

3. Enter a name and click **Create Join Token**.

![Create Join Token Modal](./images/rotate-siderolink-join-token-create-join-token.png)

4. If you are rotating the default token, set the new token as default:

    a. Click **...** next to the token
    
    b. Select **Make Default**

![Make Join Token Default](./images/rotate-siderolink-join-token-machine-join-tokens-make-default.png)

### Revoke the old join token

To revoke an old join token using the Omni UI:

1. Click **...** next to the token you want to revoke.

2. Select **Revoke**.

<Danger> If warnings are shown and you proceed anyway, the listed machines will be disconnected after the next restart of Omni or the affected machines. </Danger>

![Join Token Warnings](./images/rotate-siderolink-join-token-revoke-the-token-large-token.png)

3. If it is safe to proceed, Omni will show a green checkmark. Click **Revoke** in the confirmation dialog.

![Join Token confirmation Dialog](./images/rotate-siderolink-join-token-revoke-the-token-small-modal.png)

</Tab>
</Tabs>

After rotating the token, copy the new token and use it for future machine registrations.
